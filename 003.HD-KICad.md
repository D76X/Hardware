# KiCad FAQ

## How do I tell KiCad 9 that parts must be placed only on the top copper layer but not on the bottom copper layer?

To ensure parts are only on the Top Copper Layer (F.Cu) in KiCad 9, you set all pads within the Footprint Editor (FPid) to F.Cu (or whichever top layer you use) and leave B.Cu empty, preventing them from touching the bottom layer; for through-hole, this means using F.Cu pads and stitching with vias to B.Cu if needed, while for SMD, they are inherently on one side, making it a matter of footprint definition. You can also manage this per-pad in the PCB editor by pressing 'E' and changing the layer, but creating the footprint correctly is best for reuse. 

By defining your footprints correctly with pads only on F.Cu, KiCad knows these components belong to the top side, preventing accidental placement or copper pour on the bottom for those specific pins. 

> For Through-Hole Components (THT):

- Open the Footprint Editor: Find your component and open its footprint for editing.
- Select Pads: Select all the pads on the footprint.
- Edit Pad Properties: Press E (Edit) to open Pad Properties.
- Set Copper Layer: In the dialog, find the "Copper" dropdown and set it to F.Cu (Top Layer).
- Add Vias (if needed): If you need connections to the bottom layer, add vias in the PCB editor after placing the component, connecting F.Cu to B.Cu. 

> For Surface Mount Components (SMD):

- Check Footprint: SMD pads are typically defined for a single side (F.Cu or B.Cu) by default in their footprints.
- Verify in Footprint Editor: Open the footprint and ensure all pads are set to F.Cu.
- Place on Top: When placing in the PCB, ensure the component is assigned to the F.Cu layer (often the default). 

> In the PCB Editor (Manual Adjustment):

- Select Component: Click the component you want to adjust.
- Press 'E' (Edit Properties).
- Select Pads: In the dialog, go to the Pad Properties section (often under "Pad Properties" or similar).
- Change Layer: Set the Copper Layer to F.Cu for each pad.
